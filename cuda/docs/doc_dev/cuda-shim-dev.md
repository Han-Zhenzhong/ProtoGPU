# CUDA Runtime Shim（libcudart.so.12）Dev Doc（Implementation Guide）

This document is a developer-facing implementation guide for building the CUDA Runtime shim described in:
- Design: `cuda/docs/doc_design/cuda-shim-logical-design.md`
- Requirements: `cuda/docs/cuda-shim-requirement.md`

MVP goal: run `cuda/demo/demo` (clang-produced host binary) against gpu-sim with **no workload JSON**.

---

## 0) Deliverables (MVP)

1) A shared library that the host program can dynamically link as `libcudart.so.12` (Linux MVP).
2) Exported symbols required by the demo:
   - `cudaMalloc`, `cudaFree`, `cudaMemcpy`
   - `cudaLaunchKernel`
   - `cudaDeviceSynchronize`, `cudaGetLastError`, `cudaGetErrorString`
   - `__cudaRegisterFatBinary`, `__cudaRegisterFatBinaryEnd`, `__cudaRegisterFunction`, `__cudaUnregisterFatBinary`
   - `__cudaPushCallConfiguration`, `__cudaPopCallConfiguration`
3) Assets loading policy: env override else embedded JSON text.
4) Fatbin → PTX extraction for the clang format used by this repo.
5) Correct launch config handling: `grid/block/shared/stream` (push/pop fallback supported).
6) Correct argument packing: values from `kernelParams`, layout from PTX `.param`.
7) Correct stream ordering: per-stream FIFO, `cudaDeviceSynchronize` drains all.

---

## 1) Repo placement & file layout

Create a new shim implementation folder:

```text
src/cudart_shim/
  exports.cpp                  // extern "C" exported CUDA runtime symbols
  runtime_context.{h,cpp}      // singleton + init + shared state
  assets_provider.{h,cpp}      // env vs embedded
  assets_embedded.cpp          // generated by CMake
  fatbin_registry.{h,cpp}      // fatbin handle + PTX extraction
  kernel_registry.{h,cpp}      // hostFun -> {deviceName, module}
  launch_config_stack.{h,cpp}  // __cudaPush/PopCallConfiguration
  stream_scheduler.{h,cpp}     // per-stream FIFO + device sync
  device_memory.{h,cpp}        // cudaMalloc/cudaFree/cudaMemcpy
  error_state.{h,cpp}          // thread-local last error + cudaGetErrorString
  ptx_entry_validate.{h,cpp}   // cheap .entry validation (MVP)
  kernel_args_pack.{h,cpp}     // kernelParams -> gpusim::KernelArgs
```

Notes:
- Keep shim code **independent** from `src/apps/cli/` and workload JSON.
- Shim links against `gpusim_core` and reuses `gpusim::Runtime` in-memory APIs.

## 2) CMake integration (Linux MVP)

### 2.1 Add a new shared library target

Edit the root `CMakeLists.txt` to add something like:

- `add_library(gpu-sim-cudart-shim SHARED ...)`
- `target_link_libraries(gpu-sim-cudart-shim PRIVATE gpusim_core)`
- `set_target_properties(... OUTPUT_NAME cudart SOVERSION 12 VERSION 12.0)`

Key requirements:
- The installed/produced filename must satisfy the demo’s dynamic loader lookup (typically `libcudart.so.12`).
- Ensure exported symbol visibility (use `-fvisibility=hidden` + explicit exports only if desired; MVP can keep defaults).

### 2.2 Symbol versioning

The demo shows undefined symbols like `cudaMalloc@libcudart.so.12`.

Implementation options:
1) **No explicit version script (try first)**: some toolchains accept unversioned symbols for local builds.
2) **GNU ld version script**: add a `.map` file (e.g. `src/cudart_shim/libcudart.map`) and link with `-Wl,--version-script=...`.

MVP guidance:
- Start without version script.
- If loader errors mention missing versioned symbols, add a version script mapping all required exports to `libcudart.so.12`.

## 3) ABI: exported function signatures

### 3.1 C ABI rules

- All exported functions must be `extern "C"`.
- Avoid C++ exceptions crossing the ABI boundary. Catch-all inside exports and convert to CUDA error + diagnostic.

### 3.2 Header strategy

Do **not** depend on the real CUDA headers in the shim.

Instead, define minimal compatible types in `src/cudart_shim/cuda_abi_min.h`:
- `cudaError_t` (enum)
- `cudaMemcpyKind` (enum)
- `dim3` (struct of 3x `uint32_t`)
- `cudaStream_t` (opaque pointer type)

**Important**: The exact ABI must match what the clang-compiled host binary expects.

MVP validation step:
- Build the demo and inspect undefined symbol names (already in requirement doc).
- If runtime crashes or wrong calling conventions occur, inspect the host binary’s call sites and adjust types.

### 3.3 Registration ABI uncertainty

`__cudaRegisterFatBinaryEnd` signature varies across toolchains.

MVP approach:
- Implement `__cudaRegisterFatBinary` first.
- Implement `__cudaRegisterFatBinaryEnd` as the toolchain expects:
  - derive signature by linking the host binary against a tiny stub and reading linker errors, or
  - compare against known libcudart exports (e.g., `nm -D` on a real `libcudart.so.12`).

## 4) RuntimeContext & initialization

### 4.1 Singleton

`RuntimeContext::instance()` returns a singleton containing:
- `gpusim::Runtime rt`
- `std::string config_json, ptx_isa_json, inst_desc_json`
- `FatbinRegistry`, `KernelRegistry`
- `StreamScheduler`
- `DeviceMemory`

### 4.2 Asset initialization

On first API call requiring runtime execution (or on first registration), initialize:
1) Resolve assets via env/embedded provider.
2) Parse `GPUSIM_CONFIG` json text using the existing loader used by CLI (`gpusim::load_app_config_json_file` has a file API; for embedded text, add/ reuse a json-text loader if present).
3) Construct `gpusim::Runtime` with `AppConfig`.

Fail fast semantics:
- If assets are missing/unreadable/unparseable, store `fatal_init_error` in context.
- Subsequent CUDA APIs return error immediately, with a clear diagnostic.

## 5) Assets embedding implementation (CMake-generated .cpp)

### 5.1 Required embedded assets

Embed (at minimum):
- a default config json text
- PTX ISA json text
- inst_desc json text

### 5.2 Generation pattern

Recommended:
- Add a CMake step that reads these files at configure time and generates `src/cudart_shim/assets_embedded.cpp` containing raw string literals.

Example generated content:

```cpp
namespace gpusim_cudart_shim::embedded {
extern const char* kConfigJson;
extern const char* kPtxIsaJson;
extern const char* kInstDescJson;
}
```

Implementation details:
- Use `R"JSON( ... )JSON"` raw literals.
- Be careful about accidental `)JSON"` sequences (choose a unique delimiter).

## 6) Registries

### 6.1 FatbinRegistry

Responsibilities:
- Create and return an opaque module handle for each registration.
- Store extracted PTX text(s) for the module.

Data model:

```text
ModuleId = u64
FatbinModule {
  id: ModuleId
  ptx_texts: vector<string>
}
```

Exports:
- `__cudaRegisterFatBinary(fatbin)`:
  - parse + extract PTX texts
  - create module handle
  - store module in registry
  - return handle

- `__cudaUnregisterFatBinary(handle)`:
  - remove module and associated kernels

### 6.2 KernelRegistry

Responsibilities:
- Record `hostFun -> {moduleId, deviceName}`.

On `__cudaRegisterFunction(handle, hostFun, ..., deviceName, ...)`:
- resolve moduleId from handle
- store mapping

At launch time:
- `cudaLaunchKernel(func, ...)` uses `func` as hostFun key.

## 7) Fatbin → PTX extraction (clang format MVP)

### 7.1 Implementation plan

Start with:
- A minimal parser that detects and extracts ASCII PTX sections.

Practical MVP tactics:
- Search for the substring `.version` and `.target` and attempt to parse until the matching end marker.
- Extract full PTX module text (including `.entry`).

Correctness requirements:
- If extraction fails, return an error that clearly identifies:
  - API (`__cudaRegisterFatBinary`)
  - reason: unsupported fatbin format

Diagnostics:
- Add optional env var `GPUSIM_CUDART_SHIM_DUMP_PTX=<path>` to dump extracted PTX for debugging (optional but very helpful).

## 8) Launch configuration implementation

### 8.1 `cudaLaunchKernel` preferred path

Use the explicit args:
- `gridDim`, `blockDim`, `sharedMemBytes`, `stream`

Validate:
- any dimension == 0 → error + last error.

### 8.2 Push/Pop fallback

Implement a **thread-local stack**:

```text
thread_local vector<CallConfig> g_call_config_stack;
CallConfig { dim3 grid, dim3 block; size_t shared; cudaStream_t stream; }
```

- `__cudaPushCallConfiguration(...)` pushes.
- `__cudaPopCallConfiguration(...)` pops.

When used:
- Some compiler paths call push, then later `cudaLaunchKernel` with default dims.
- For MVP, if `cudaLaunchKernel` sees “default/zero dims” it may consult pop (only if required by observed host behavior).

---

## 9) StreamScheduler (FIFO ordering)

### 9.1 Stream representation

- Default stream: `stream == nullptr`.
- Maintain map: `cudaStream_t -> StreamState`.

MVP simplifying assumption:
- Execute synchronously under a global lock.

`cudaDeviceSynchronize()`:
- drains all stream queues (or no-op if synchronous execution is used).

---

## 10) DeviceMemory

### 10.1 `cudaMalloc` / `cudaFree`

- Use `gpusim::Runtime::device_malloc(bytes, align)`.
- Return the resulting `DevicePtr` encoded as `void*`.
- Track allocation metadata in `alloc_map` keyed by `DevicePtr`.

### 10.2 `cudaMemcpy`

Support kinds:
- H2D, D2H, D2D

Implementation sketch:
- Decode device pointers:
  - `DevicePtr p = static_cast<DevicePtr>(reinterpret_cast<uintptr_t>(devPtrVoid))`
- Validate allocation existence and bounds.
- Use gpu-sim runtime copy helpers.

Note:
- gpu-sim currently has host buffers (`host_alloc`, `host_write`, `host_read`) and runtime memcpy helpers; pick the simplest path that avoids unnecessary host-buffer allocations.

## 11) Kernel launch pipeline

### 11.1 Entry selection

Source of truth:
- `deviceName` recorded during `__cudaRegisterFunction`.

Validate `.entry deviceName` exists:
- MVP: string search in PTX text.
- Preferred: attempt to bind kernel by name using gpu-sim Binder (if bind fails → entry not found).

### 11.2 Pack args: `kernelParams` → `gpusim::KernelArgs`

Steps:
1) Parse PTX text tokens: `Parser::parse_ptx_text_tokens(ptx_text, "<fatbin>")`.
2) Bind kernel: `Binder::bind_kernel_by_name(mod_tokens, deviceName)`.
3) `KernelTokens.params` becomes `KernelArgs.layout`.
4) Allocate blob length: `max(offset+size)`.
5) For each param in order:
   - if param is pointer (`u64`):
     - read `void* host_devptr = *reinterpret_cast<void**>(kernelParams[i])`
     - convert to `DevicePtr` value (numeric)
     - validate it exists in `alloc_map`
     - write little-endian u64 into blob
   - else:
     - copy `param.size` bytes from `kernelParams[i]` into blob at `param.offset`

Mismatch handling:
- If missing argument, unsupported param type, or invalid pointer: set last error and return an error.

### 11.3 Execute via gpu-sim Runtime

Use:
- `gpusim::Runtime::run_ptx_kernel_with_args_text_entry_launch(ptx_text, ptxIsaJson, instDescJson, deviceName, args, launch)`

Launch config:
- `gpusim::LaunchConfig` contains grid/block/warp_size.
- warp_size comes from sim config.
- sharedMemBytes must be recorded/logged for MVP; follow-up is threading it into gpu-sim core.

## 12) ErrorState

Requirements:
- Per-thread last error compatible with `cudaGetLastError`.
- `cudaGetErrorString` supports at least errors produced by the shim.

Implementation:

```text
thread_local cudaError_t g_last_error;

cudaGetLastError():
  ret = g_last_error; g_last_error = cudaSuccess; return ret
```

Diagnostics:
- On fail-fast conditions, print:
  - API name
  - deviceName (if available)
  - classification: missing registration / fatbin parse / entry missing / arg pack / memcpy invalid

## 13) Tests & validation

### 13.1 Unit tests (recommended additions)

Add new test executables under `tests/` for:
- fatbin parser: can it extract PTX from a saved demo fatbin blob?
- kernel args packer: given PTX text + dummy kernelParams, produce expected blob.
- launch config stack: push/pop correctness and empty-pop behavior.

### 13.2 Integration: run the demo

Linux MVP run procedure:
1) Build `gpu-sim-cudart-shim`.
2) Ensure the loader uses it:
   - set `LD_LIBRARY_PATH` to include the build output directory, or
   - link demo with rpath pointing to it.
3) Run `cuda/demo/demo`.
4) Verify:
   - kernel executes
   - memcpy results correct
   - `cudaGetLastError` returns success

## 14) Implementation milestones (recommended order)

1) CMake target + export stubs returning `cudaErrorNotSupported` (or similar) with last-error.
2) AssetProvider + RuntimeContext init.
3) `cudaMalloc/cudaFree/cudaMemcpy` + allocation validation.
4) Registries: `__cudaRegisterFatBinary*`, `__cudaRegisterFunction`, `__cudaUnregisterFatBinary`.
5) Fatbin→PTX extraction for demo.
6) Launch config: push/pop + `cudaLaunchKernel`.
7) Entry validation + args packer.
8) Stream FIFO model + `cudaDeviceSynchronize`.
9) Polish diagnostics + add tests.
