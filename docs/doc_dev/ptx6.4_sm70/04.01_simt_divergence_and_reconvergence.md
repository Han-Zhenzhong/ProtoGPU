## 0) 当前现状（为什么需要 M5）

现状（M4）
- SIMT 编排层在 `src/simt/simt.cpp` 对 `BRA/RET` 做 uniform-only 检测：
  - `m = warp.active & uop.guard`
  - `m` 非空且 `m != warp.active` → `simt/E_DIVERGENCE_UNSUPPORTED`（fail-fast）

M5 目标
- 移除该 fail-fast：允许 `taken` 与 `fallthrough` 同时存在。
- 以 SIMT stack 执行两个路径并在 `reconv_pc` 合并。
- `ret` 改为 lane-wise 退出（避免一个 lane 返回导致整个 warp.done）。

---

## 1) 实现边界（冻结）

- 仍采用 **warp 单 PC + active_mask + stack**（不引入 per-lane PC）。
- 仍保持 inst 级 commit：`next_pc`/PC 更新在一条 inst 的所有 uops 执行结束后生效。
- Units 不负责分歧判定/合流编排；该职责属于 SIMT executor。

## 3) M5.1：CFG + ipdom（reconv_pc 预计算）

### 3.1 目的

divergence 的 join 点不能靠 `pc+1` 推导；必须对每条“可能分歧的 `bra`”预先得到 `reconv_pc`：

- `reconv_pc[pc] = ipdom(pc)`（immediate post-dominator）

### 3.2 CFG 构建（落点建议）

建议放在 `src/simt/control_flow_analysis.*`。

节点
- `0..N-1` 对应 inst index。
- `EXIT` synthetic 节点（index = N）。

successors 规则（以 InstList index 表达）
- 普通指令：`pc -> pc+1`（若 pc+1 < N，否则 `pc -> EXIT`）
- `ret`：`pc -> EXIT`
- `bra target`：
  - unconditional：`pc -> target`
  - predicated（有 `InstRecord.pred`）：`pc -> target` 与 `pc -> pc+1`

注：Binder 阶段已把 label 绑定为 imm(pc)，因此 `bra` 的 target 应可直接从 `InstRecord.operands[0].imm_i64` 获得。

### 3.3 ipdom 算法（建议实现）

采用经典迭代式 post-dominator：

- 初始化：
  - `pdom(EXIT) = {EXIT}`
  - 其他节点：`pdom(n) = AllNodes`
- 迭代：
  - `pdom(n) = {n} ∪ (⋂_{s∈succ(n)} pdom(s))`
- 收敛后求 `ipdom(n)`：
  - `ipdom(n)` 是 `pdom(n) - {n}` 中“最接近 n 的那个”（定义为：其 pdom 集合在候选中最小/最具体）。

输出
- `reconv_pc_by_pc[pc] = ipdom(pc)`（仅对 predicated `bra` 需要；对 unconditional 可留空）。

失败策略
- 若某个 predicated `bra` 在收敛后没有可用的 ipdom（或算出 EXIT 但你不想允许），则 SIMT 执行前 fail-fast：
  - `Diagnostic{ module="simt", code="E_RECONV_MISS" }`

并行注意点
- `SimtExecutor::run` 可能在多 SM worker 线程里跑 CTA。建议在进入 CTA 循环前对 `kernel` **预计算一次**并共享只读结果。

## 4) M5.2：SIMT stack（split/merge）落地到 `src/simt/simt.cpp`

### 4.1 数据结构（最小化）

```text
SimtFrame { pc, active, reconv_pc }
```

WarpState 增量
- `exited: LaneMask`（初始化为 0）
- `simt_stack: vector<SimtFrame>`

初始化
- 原先 `warp.pc` / `warp.active` 的初始化保留，但成为 stack 的初始 frame：
  - `push({pc=0, active=active0, reconv_pc=EXIT})`

来源真相
- 执行时的“当前 pc/active”统一取自 `simt_stack.back()`。

### 4.2 step 前规范化（必须有，防止死循环）

在每次 fetch 前执行：
1) `frame.active &= ~warp.exited`
2) `frame.active == 0` → pop
3) `frame.pc == frame.reconv_pc` → pop 并尝试 merge
4) stack 为空 → warp 完成

merge 最小规则
- 如果 pop 后 top 的 `pc == popped.pc`：`top.active |= popped.active`
- 否则将 popped 作为“等待帧”保留（实现上可直接 push 回去，但要避免立刻再次触发 `pc==reconv` 的 pop/merge 死循环；建议引入 Join frame 或在规范化阶段区分处理）。

### 4.3 `BRA` 的执行策略（推荐：SIMT 接管 split）

现状（M4）
- SIMT 做 uniform-only 检测后调用 `ControlUnit::step` 得到 `next_pc`。

M5 建议
- 对 `uop.op == BRA`：由 SIMT 直接读取 target_pc（并复用 ControlUnit 的 operand/arity 校验逻辑，或在 SIMT 复刻同样的 diag）。
- 计算：
  - `taken = frame.active & uop.guard`
  - `fallthrough = frame.active & ~uop.guard`

分三类：
- `taken` 空：不跳转（commit 走 `pc+1`）
- `fallthrough` 空：uniform taken（commit 走 target_pc）
- 两者都非空：divergence
  - `reconv_pc = reconv_pc_by_pc[pc]`（来自 M5.1）
  - 改写当前 top 为 fallthrough frame（`pc+1` + `fallthrough` + reconv_pc）
  - push taken frame（`target_pc` + `taken` + reconv_pc）

commit 语义
- divergence 情况下，`BRA` 本身不需要依赖 `StepResult.next_pc` 通道；下一步 fetch 直接从新 top frame 取 pc。

诊断
- 若 divergence 但找不到 reconv_pc：`simt/E_RECONV_MISS`（fail-fast）

### 4.4 可观测（建议最小加点）

现有 `Event` 支持 `extra_json`，不破坏对外 contract。建议在 split/merge 处 emit Ctrl 类事件：
- category=`Ctrl`, action=`SIMT_SPLIT`/`SIMT_MERGE`/`SIMT_POP`
- `pc` 填分歧点/合流点；lane_mask 可填“当前 active”；细节放入 `extra_json`。

## 5) M5.3：lane-wise `RET`（避免 warp-wide done）

现状（M4）
- `ControlUnit::step(RET)` 直接 `warp.done = true`。

M5 冻结语义
- `RET` 对 `exec_mask` 生效：
  - `warp.exited |= exec_mask`
  - `frame.active &= ~exec_mask`
- frame active 变空后由规范化逻辑 pop。
- warp 完成条件：stack 为空（或 exited 覆盖初始 active0 且无待执行 frame）。

实现建议（尽量少改 contracts）
- 在 SIMT dispatch control uop 时，对 `RET` 做 special-case：
  - 计算 `exec_mask = frame.active & uop.guard`
  - 更新 exited/mask
  - 不调用 `ControlUnit::step(RET)`（避免它写 `warp.done`）

（如果坚持继续走 ControlUnit，则需要扩展 `StepResult` 表达“哪些 lanes 退出”，这会牵涉 contracts 与所有调用点，代价更大。）

## 6) 验收（建议新增 targeted tests）

现有 targeted tests：`tests/simt_predication_controlflow_tests.cpp` 已覆盖：
- predication 注入越界（`E_PRED_OOB`）
- uniform-only divergence fail-fast（`E_DIVERGENCE_UNSUPPORTED`）
- `E_NEXT_PC_CONFLICT`、`E_PC_OOB`

M5 建议新增用例（同风格新增函数即可）

1) `test_divergence_if_else_join_completes`
- warp_size=2
- setp 产生 lane0 true / lane1 false
- `@p bra L_then;` else 块执行 `mov r0, 1; bra L_join;` then 块执行 `mov r0, 2;` `L_join: ret;`
- 断言：completed=true 且最终 lane0/lane1 的 r0 分别为 2/1。

2) `test_divergent_ret_lane_wise`
- lane0 早退，lane1 继续执行写寄存器后再退。
- 断言：lane1 的后续写入未被 lane0 的 ret 影响。

3) `test_reconv_missing_is_fail_fast`
- 构造一个 predicated `bra` 但让 reconv 预计算返回空（例如人为注入/禁用分析）
- 断言：`simt/E_RECONV_MISS`。

说明
- 如果短期内不想实现完整 ipdom，可先在 test kernel 上“硬编码 reconv_pc map”作为 bring-up（但必须在 doc 里明确这是临时策略，最终仍要落回 CFG+ipdom）。
