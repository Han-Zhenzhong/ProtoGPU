# 00.01 Fail-fast Memory OOB Dev Plan（global）

目标：把“global OOB/未分配必须报错（禁止静默读 0 / 禁止静默写成功）”冻结成可执行开发清单，并用稳定 fixture + Tier‑0 回归锁死行为。

单一真源
- 语义与边界：[docs/doc_dev/ptx6.4_sm70/00.01_fail_fast_memory_oob.md](../../../doc_dev/ptx6.4_sm70/00.01_fail_fast_memory_oob.md)
- Tier‑0 闸门工作流（回归与回溯地图）：[docs/doc_dev/ptx6.4_sm70/00_tier0_gate_workflow.md](../../../doc_dev/ptx6.4_sm70/00_tier0_gate_workflow.md)

---

## 1) 冻结对外观测结果（必须满足）

当发生以下情况之一：
- 访问未分配 global 区间
- 访问越界 global 区间

必须满足：
- `out.sim.completed == false`
- `out.sim.diag.has_value() == true`
- 诊断码：
  - 对齐但越界/未分配：`units.mem/E_GLOBAL_MISS`
  - 不对齐访问：`units.mem/E_GLOBAL_ALIGN`

---

## 2) 双层防线（强制）

### 2.1 内存模型层：allocation-range 检查

落点
- 接口：`include/gpusim/memory.h`
- 实现：`src/memory/memory.cpp`

强制约束
- `AddrSpaceManager::read_global(addr, size)`：越界/未分配必须返回 `std::nullopt`
- `AddrSpaceManager::write_global(addr, bytes)`：越界/未分配必须返回 `false`

### 2.2 执行层：miss/align 转译为诊断

落点
- `src/units/mem_unit.cpp`

强制约束
- 对齐失败：`units.mem/E_GLOBAL_ALIGN`
- read/write miss：`units.mem/E_GLOBAL_MISS`
- 一旦产生诊断：必须 fail-fast（本次 run 终止，不可“继续执行”）

---

## 3) 资产与回归（锁死行为）

### 3.1 Fixture

- Fixture：`tests/fixtures/ptx/mem_oob_st_global_f32.ptx`
- 约束：用 `[%rd + imm]` 直接制造越界地址，避免依赖 bring-up 未纳入的 `add.u64 (reg,reg,imm)` 等写法。
- 推荐：让 fixture 保持“对齐但越界”，避免被 `E_GLOBAL_ALIGN` 抢先触发。

补充 fixtures（把 MISS 与 ALIGN 分开锁死）
- 未分配但对齐：`tests/fixtures/ptx/mem_unalloc_ld_global_f32.ptx` → 期望 `E_GLOBAL_MISS`
- 不对齐：`tests/fixtures/ptx/mem_misaligned_ld_global_f32.ptx` → 期望 `E_GLOBAL_ALIGN`

### 3.2 测试断言

- 回归：`tests/tiny_gpt2_minimal_coverage_tests.cpp`
  - `test_oob_st_global_must_fail_fast()`：期望 `E_GLOBAL_MISS`
  - `test_unalloc_ld_global_must_fail_fast()`：期望 `E_GLOBAL_MISS`
  - `test_misaligned_ld_global_must_fail_fast()`：期望 `E_GLOBAL_ALIGN`

### 3.3 Merge gate

- 默认纳入 Tier‑0：`gpu-sim-tiny-gpt2-mincov-tests`
- 命令：

```bash
ctest --test-dir build -C Release -V -R "^gpu-sim-tiny-gpt2-mincov-tests$"
```

---

## 4) 未来扩展的红线

- 新增 memcpy/engine/并行路径时：不得绕过 allocation-range 检查与执行层诊断转译。
- 任何“返回 0 代替错误”的行为都视为回归（必须被测试捕获）。
